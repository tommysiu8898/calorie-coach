## 🔧 Fix the EAS build error

### Step 1: Ensure `expo` and `expo-modules-autolinking` are installed in the correct workspace

In your Replit terminal, run:

```bash
cd ~/workspace/artifacts/calorie-tracker
pnpm add expo expo-modules-autolinking
```

If you are using a monorepo, also run from the root:

```bash
cd ~/workspace
pnpm add -w expo-modules-autolinking
```

### Step 2: Create a proper `metro.config.js` in the calorie-tracker workspace

EAS needs Metro config to resolve modules correctly. Create or edit `/home/runner/workspace/artifacts/calorie-tracker/metro.config.js`:

```javascript
const { getDefaultConfig } = require('expo/metro-config');
const path = require('path');

const config = getDefaultConfig(__dirname);

// For monorepo support
config.watchFolders = [path.resolve(__dirname, '../..')];
config.resolver.nodeModulesPaths = [
  path.resolve(__dirname, 'node_modules'),
  path.resolve(__dirname, '../../node_modules'),
];

module.exports = config;
```

### Step 3: Update `eas.json` to explicitly set the project root

In your `eas.json` (the one you already have), add a `"cli"` section with `"projectRoot"` if needed, but more importantly, set `"expo"` autolinking. Alternatively, add a hook to run `expo prebuild` if necessary.

But the cleaner fix: ensure the `expo` package is properly hoisted. In your root `package.json`, add:

```json
"pnpm": {
  "overrides": {
    "expo-modules-autolinking": "latest"
  }
}
```

Then run `pnpm install` again.

### Step 4: Commit and push all changes to GitHub

Use Replit's **Version Control** panel (left sidebar, Git icon) to commit and push the updated `metro.config.js`, `package.json`, and lockfile.

If you still have trouble with `git push`, use the Replit Git panel – it should be already connected. Click **"All changes"**, write a commit message, then click **"Push"**.

### Step 5: Trigger a new EAS build

After pushing, run:

```bash
eas build --platform ios --profile internal-test
```

When prompted about the provisioning profile, choose **"No, let me choose devices again"** and select **all devices** (including the new UDID you added in Apple Developer). The build will then create an updated IPA for all 100 devices.

Then try the build again.
