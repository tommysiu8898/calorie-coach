We need to fix the coach chat in the React Native (Expo) + Node.js app. The UI now has **tip buttons** (e.g., "Nearby Gyms", "Nearby Restaurants", "Nearby Health Clinics") instead of relying on natural language for place search. The backend should handle explicit category requests.

**Backend endpoint**: `POST /api/chat/coach`

**Expected request body**:
{
  "message": string,          // user's typed message (could be empty if triggered by button)
  "category": "gym" | "restaurant" | "health" | null,   // from tip button (preferred)
  "lat": number,
  "lng": number,
  "appLanguage": "en" | "zh"
}

**Logic**:
1. If `category` is provided (non-null), directly use it for Mapbox search – **skip intent detection**.
2. If `category` is null, fall back to simple keyword detection (for backward compatibility with typed messages).
3. Use Mapbox Search Box API category endpoint:
   - URL: `https://api.mapbox.com/search/searchbox/v1/category/{category}`
   - Parameters:
     - `proximity` = `${lng},${lat}` (longitude first!)
     - `limit` = 10
     - `radius` = 5000
     - `language` = "zh" if appLanguage === "zh" else "en"
     - `access_token` = process.env.MAPBOX_ACCESS_TOKEN
4. Map categories to Mapbox supported types:
   - "gym" → "gym"
   - "restaurant" → "restaurant"
   - "health" → "health" (or "clinic" if "health" not supported; test and adjust)
5. Return unified JSON:
   {
     "success": true,
     "reply": string,   // e.g., "Here are 10 gyms near you" (localized)
     "places": [ { name, address, distance, coordinates, mapbox_id } ]
   }
6. Error handling: timeout 5 seconds, friendly message if no results.
