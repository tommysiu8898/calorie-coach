看到了，这两个错误其实很常见，一个和前端的位置权限请求有关，另一个是后端调用Mapbox API时参数用错了。我们一步一步来修复它们。

### 🐞 错误1：“Location access is needed”

这个问题通常不是用户没给权限，而是**App没有在用户允许权限后，去重新获取位置**。

你可以试一下这两个修复方法：
*   **请求成功后调用位置获取函数**：不要在用户允许权限后就结束，而是立刻调用一个 `getLocation()` 函数来主动获取位置。
*   **增加一个备选方案**：为了防止 `expo-location` 在某些环境下不稳定，可以使用浏览器默认的 `navigator.geolocation` API 作为备选，以提高兼容性。

### 🐞 错误2：“unknown field `session_token`”

这个400错误的原因找到了，是代码向 **`/category` 端点传递了 `session_token` 参数，但这个参数压根不属于它**。`session_token` 只用在两步交互式搜索 (`/suggest` 和 `/retrieve`) 来计费，而 `/category` 是一次性查询，不支持这个字段。

请检查后端调用代码，务必**删除 `session_token` 参数**。

#### 1. 修复后的前端位置请求逻辑
```typescript
// 修复后的前端位置请求函数
const requestLocationPermission = async () => {
  let { status } = await Location.requestForegroundPermissionsAsync();
  if (status !== 'granted') {
    // 如果权限被拒绝，可以再尝试用浏览器定位作为备用
    if (navigator.geolocation) {
      navigator.geolocation.getCurrentPosition(
        (position) => {
          setUserLocation(position.coords);
        },
        (error) => console.log('备用定位失败: ', error)
      );
    }
    return;
  }
  // 关键修复：权限获得后，立刻主动获取一次位置
  let location = await Location.getCurrentPositionAsync({});
  setUserLocation(location.coords);
};
```

```python
# 修复后的后端调用示例 (Python)
import requests

# 当用户点击 "附近健身房" 按钮时，调用 /category 端点
def search_nearby_gyms(lat, lng, app_language):
    url = "https://api.mapbox.com/search/searchbox/v1/category/gym"
    params = {
        "proximity": f"{lng},{lat}", # 注意经度在前
        "limit": 10,
        "radius": 5000,
        "language": "zh" if app_language == 'zh' else "en",
        # 再次强调，千万不能加 session_token 参数！
        "access_token": "你的_mapbox_公钥"
    }
    
    response = requests.get(url, params=params)
    if response.status_code == 200:
        return parse_results(response.json())
    else:
        print(f"请求出错: {response.status_code}")
        return []

def parse_results(data):
    """解析返回的JSON数据，提取需要的信息"""
    places = []
    for feature in data.get('features', []):
        places.append({
            'name': feature['properties']['name'],
            'address': feature['properties'].get('address', '地址未知'),
            'coordinates': feature['geometry']['coordinates']
        })
    return places
```
